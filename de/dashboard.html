Dashboard

Dashboard bietet, im Gegensatz zu “Messages", eine grafische Benutzeroberfläche. Es können universelle Anzeige- und Steuerelemente angelegt werden, wie z. B. eine Temperaturanzeige oder ein Lichtschalter (Verwendung des MQTT-Protokolls).

  • Tippen Sie nach dem Starten der App auf den Listeneintrag eines Push-Server-Kontos.
  • Tippen Sie dann im Fenster „Messages“ auf das Icon .

Die Ansicht „Dashboard“ öffnet sich. Hier lassen sich Komponenten zur Anzeige von Sensordaten bzw. Werten des spezifizierten Topics anlegen und/oder zur Eingabe von Werten (über entsprechende Steuerlelemente), die an das angegebene „Publish Topic“ gesendet werden.

  • Tippen Sie auf das Icon .

Ein Einblendmenü öffnet sich und bietet verschiedene Komponentenvorlagen an, welche für die unterschiedlichsten Zwecke eine gute Ausgangsbasis für eigene Lösungen bieten.

KomponenteErläuterung
Text ViewLiefert den Inhalt einer Nachricht in Textform. Siehe dazu den Abschnitt Text View

Vorlagen für Komponenten

Allgemeine Konfigurationselemente (in mehreren Komponenten verwendet):
  • Name
    Hier geben Sie der anzulegenden Komponente einen Namen
  • Group
    Hier gruppieren Sie die Komponente in eine (bestehende) Gruppe
  • Position
    Hier legen Sie die Position der Komponentenkachel innerhalb einer Gruppe fest
  • Color
    Hier können Sie die Farbe für Vodergrund, Hintergrund und Text auswählen
  • Text Size
    Hier können Sie die verwendete Textgröße („small“/„medium“/„large“) auswählen
  • Image
    Hier können Sie ein Bild auswählen oder ein eigenes Bild importieren
  • Topic (sub)
    Hier geben Sie den Verzeichnispfad für die gewünschte Registrierung (Subscribe) an
  • Filter Script
    Hier haben Sie die Möglichkeit, JavaScript-basierte Filterskripte einzugeben, die Benachrichtigungen analysieren und formatieren, um die gewünschten Daten in Form einer Textnachricht darzustellen xxx
  • Topic (pub)
    Hier geben Sie den Verzeichnispfad für die gewünschte Veröffentlichung (Publish) an
  • Retain
    Hier können Sie festlegen, dass Benachrichtigungen auf dem MQTT-Server vorgehalten werden
  • Script
    Hier können Sie ein Skript eintragen, welches bei „Topic (pub)“ ausgeführt werden soll, also beim Betätigen der Schaltfläche xxx

Text View

Stellt den Inhalt einer Nachricht in Textform dar. Sofern ein „Publish Topic“ angegeben wurde, kann in der Detailansicht ein Wert eingegeben und gesendet werden.

Zusätzliches Konfigurationselement:
  • Input Type
    Hier können Sie von Text- auf Zifferneingabe umschalten

Button/Switch

Erzeugt eine Schaltfläche, welche z. B. als Taster fungiert, der bei Betätigung eine Nachricht versendet. Dazu wird in „Off state“ nichts eingetragen. Wird die Schaltfläche als Schalter angelegt, der in Abhängigkeit des Wertes der letzten empfangenen Nachricht seinen Zustand ändert (an/aus), muss im „Off state“ ein Topic eingetragen sein.

Beispiel:

Eine Komponente soll konfiguriert werden, die eine Lampe ein- und ausschaltet und gleichzeitig über den aktuellen Schaltstatus informiert.

  • Wählen Sie aus dem Menü den Eintrag „Button/Switch“ aus:

  • Legen Sie zuerst einen Namen für die Komponente fest (im obigen Beispiel haben wir sie „Schalter“ genannt).
  • Definieren Sie ggf. die Hintergrundfarbe der Komponentenkachel an sich sowie die Vorder- und Hintergrundfarbe der Schaltzustände An („On state“) bzw. Aus („Off state“).

Zu einer Hintergrundfarbe lässt sich zusätzlich auch ein Bild verwenden:

  • Tippen Sie auf die Schaltfläche neben „Image“.

Es öffnet sich das Fenster „Select an Image“, welches eine Reihe von Icons und Symbolen zur Verfügung stellt. Alternativ dazu können auch eigene Bilder verwendet werden, die zuvor in die App importiert wurden (Eigene Bilder in die App importieren).

  • Tragen Sie sowohl für den Empfang („Topic (sub)“) als auch für das Senden („Topic (pub)“) ein Topic ein.

Nur so kann die Schaltkomponente den Schaltstatus („Topic (sub)“) vom Server abfragen und Schaltbefehle an den Server geben („Topic (pub)“).


Zusätzliche Konfigurationselemente:
  • On State
    Hier tragen sie einen Wert für den Schaltzustand An ein, z. B. „On“
  • Off State
    Hier tragen sie einen Wert für den Schaltzustand Aus ein, z. B. „Off“

Progress Bar/Slider

Ein Wertebereich wird durch einen Balken dargestellt, wobei der Balken bis zum aktuellen Wert eingefärbt wird (wie bei einer Fortschrittsanzeige). Wurde ein „Publish Topic“ angegeben, kann in der Detailansicht durch einen Schieberegler ein neuer Wert gesetzt und gesendet werden.

Zusätzliche Konfigurationselemente:
  • Range
    Hier definieren Sie den Wertebereich eines Fortschrittsbalkens
  • Display in percent
    Hier können Sie festlegen, dass ein Wert in Prozent (bezogen auf den gesamten Wertebereich) angezeigt wird
  • Decimal
    Hier können Sie die Nachkommastellen des angezeigten Wertes festlegen

Option List

Erlaubt die Definition einer Menge von Werten. Für jede Option kann ein Anzeigewert definiert werden, der anstatt des tatsächlichen Wertes angezeigt wird.

Group View

Für eine bessere Übersicht im Dashboard lassen sich Gruppen anlegen, denen einzelne Komponenten zugeordnet werden können.

Hinweis:

Einzelne Komponenten lassen sich gruppieren oder können auch einer anderen Gruppe zugeordnet werden. Dazu tippen und halten Sie die gewünschte Komponentenkachel („Long Press“), bis ein erscheint und Sie per in den Bearbeitungsmodus wechseln können.

Über „Group“ und „Pos“ kann das Icon nun in die gewünschte Position gebracht werden.

Custom View

Hier lassen sich beliebige Komponenten in HTML anlegen. Dazu kann der HTML-Code direkt in das dafür vorgesehene Feld geschrieben oder kopiert werden. Alternativ erhalten Sie durch Tippen auf das Überlaufmenü () unter „Insert Example“ drei verschiedene HTML-Vorlagen (Basic HTML, Color Picker und Gauge), die sich direkt in die Komponente einfügen und bearbeiten lassen.

Zusätzliches Konfigurationselement:
  • HTML
    Hier können Sie Ihren eigenen HTML-Code zur Gestaltung der Komponentenkachel eingeben. Dabei ist auch die Einbindung von JavaScript zulässig

Eigene Bilder in die App importieren

Einige Dashboard-Komponenten erlauben die Einbindung von Bildern (Hintergründe, Status von Schaltern usw.). Damit eigene Bilder in der App zur Verfügung stehen, müssen diese zuvor in die App importiert werden:

  • Wählen Sie dazu im Fenster „Dashboard“ den Eintrag „Manage Images“ aus dem Überlaufmenü () aus.

  • Öffnen Sie dann im Fenster „Manage Images“ erneut das Überlaufmenü (), tippen Sie auf „Import“ und wählen Sie die Bilder aus, die Sie in die App importieren möchten.

Alle in die App importierten Bilder landen im Fenster „Manage Images“:

Jedesmal, wenn die App beendet wird, werden Bilder, die in keiner Komponente verwendet werden, gelöscht. Um dies zu verhindern, können sie geschützt werden. Im obigen Bild sind bereits zwei zuvor importierte Bilder zu sehen. Beide sind mit solch einem Schutz () versehen.

  • Tippen und halten Sie die gewünschte Bilddatei („Long Press“), bis das Schlosssymbol erscheint.
Die mit einem versehenen Bilder bleiben beim Neustart der App erhalten.
Hinweis zu Bildern mit Transparenzen:

Werden Bilder, welche Transparenzen enthalten, über das Feld „Image“ in die Komponente eingebunden, so bleiben diese Transparenzen erhalten.
